BIDS is a structure
BIDS is not a new file format
BIDS is a standard
BIDS is not a software tool. However, there is a large and growing amount of tools that are compatible with it that ease data archiving, data discovery/search, and analysis
There is no "one" BIDS
BIDS exist for (a growing amount of) different modalities. There is constant (open!) development of all of them
  • What is BIDS

    • BIDS helps you and others to intuitively understand your data
    A consensus on data organization spares you and others time to dive in to or rearrange data or scripts
    BIDS opens up a large range of tools for you:
    ⮊ BIDS Apps, e.g., fmriprep or MRIQC
    ⮊ BIDS-aware tooling, e.g., PyBIDS
    ⮊ brainlife.io
    ⮊ Upload your (or download others) BIDS-compliant datasets from OpenNeuro

    Trinity of research data handling

    $HOME (laptop, workstation)
    • fully flexible/undefined structure
    • limited storage
    • minor computational resources
    $COMPUTE (cluster, HPC, cloud)
    • analysis-defined structure
    • analysis-appropriate resources
    $DATA
    • fully-determined, domain-agnostic structure
    • "unlimited" storage
    • managed backups
    Note to self

    Research data flow

    • Researchers are free to do whatever at $HOME (within limits)
    • $COMPUTE pulls data (exclusively) from $DATA
    • Results from $COMPUTE are immediately/eventually pushed back to $DATA
    Impact of ephemeral $HOME and $COMPUTE
    • No limits to tinkering and exploration of data
    • Anything that needs to scale requires researchers to use $COMPUTE and thereby $DATA
    • Anything that is/was relevant for computation is tracked in $DATA
    • The only trusted place for any data is $DATA, hence anything that could be important is there

    Researchers' point of view (1)

    Create a new dataset for data, analysis results, manuscripts, etc. 
    
% datalad create -c inm7
    • inm7 is institute-specific configuration procedure that performs all relevant setup
    • fully configured link to $DATA
    • creator's identity recorded in dataset
    Note to self

    Researchers' point of view (2)

    Sync with $DATA for backup, collaboration, or computation 
    
% datalad publish --to inm7
    • pushes all changes to the dataset repository
    • transfers data files
    • dataset now exists in a homogeneous, accessible, domain-agnostic format that links its evolution to individual people, and all associated data files

    Researchers' point of view (4)

    Use a dataset for analysis, manuscript writing, etc. 
    
% datalad create -c inm7 mynewdataset
% datalad install --dataset mynewdataset
                  --source <ID/URL>
                  mynewdataset/inputs/...
    • dataset linkage is established
    • input and output datasets remain separate, modular entities
    • on publish linkage information becomes accessible in $DATA
